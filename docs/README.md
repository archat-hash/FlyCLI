# 🚁 FlyCLI: Твій дрон під повним контролем (та AI)

**FlyCLI** — це потужний консольний інструмент на базі Node.js, створений для надійної взаємодії з польотними контролерами Betaflight. 

Цей інструмент спроектовано спеціально для **AI-агентів** (ChatGPT, Claude, Gemini) та розробників, яким потрібен стабільний, автоматизований доступ до CLI без графічного інтерфейсу.

---

## 🌟 Чому FlyCLI?

- **Подієва архітектура (Event-Driven)**: Забудьте про нестабільні тайм-аути. FlyCLI використовує асинхронні ланцюжки очікування подій (Wait Chains), що робить зв'язок наднадійним.
- **Автовизначення (Auto-Discovery)**: Розумне сканування портів підкаже, де саме підключено ваш Betaflight пристрій.
- **AI-Ready**: Вивід у JSON дозволяє ШІ-агентам легко парсити стан дрона, виявляти несправності та вносити зміни.
- **Handshake-валідація**: Повна імітація протоколу підключення Betaflight Configurator (MSP + CLI banner).

## 🚀 Швидкий старт

### 1. Встановлення
```bash
git clone https://github.com/oleh/FlyCLI.git
cd FlyCLI
npm install
```

### 2. Пошук пристрою
Команда `scan` автоматично знайде ваш польотний контролер:
```bash
./index.js scan
```
**Вивід:**
```
🚀 [Lately Betaflight] /dev/tty.usbmodem0x80000001
   Manufacturer: Betaflight
   VID: 0483 PID: 5740
```

### 3. Виконання команд
```bash
./index.js execute /dev/tty.usbmodem0x80000001 115200 "status"
```

## 🤖 Використання з AI-агентом

FlyCLI ідеально підходить для того, щоб ШІ налаштував ваш дрон або провів діагностику.

### Порада для користувача:
Просто надайте агенту доступ до цієї папки та скажіть:
> *"Я підключив дрон. Використовуй FlyCLI, щоб перевірити статус моторів та надати мені звіт про можливі проблеми."*

### Інструкція для AI-агента:
1. Використовуй `./index.js scan --json` для ідентифікації порту.
2. Використовуй `./index.js execute <port> 115200 "<команда>" --json` для взаємодії.
3. Парси JSON-результат для отримання чистих даних без зайвого шуму консолі.

---

## 🛠 Команди CLI

| Команда | Опис |
| --- | --- |
| `scan` | Пошук доступних портів з маркуванням Betaflight |
| `execute <port> <baud> <cmd>` | Виконання будь-якої CLI команди (diff, dump, set, save...) |
| `--json` | Флаг для отримання результату в структурованому форматі |

## ⚠️ Важливо
- Завжди робіть `diff all` перед внесенням змін.
- Команда `save` призводить до **перезавантаження контролера** — це нормально.
- ⚠️ **УВАГА**: Команда `defaults` на платах **STM32F411 Black Pill** може призвести до "цеглування" (втрати зв'язку по USB). Це стається через те, що дефолтні налаштування прошивки можуть вимкнути USB VCP або змінити мапування портів. Будьте готові до перепрошивки через DFU, якщо ви виконаєте цю команду на цій платі.
- Не підключайте пропелери під час роботи з USB.
