# 🚁 FlyCLI

**FlyCLI** — консольний інструмент для керування польотними контролерами Betaflight з комп'ютера або через AI-агента.

> **Ідея проста**: підключаєте контролер по USB → запускаєте команду → отримуєте інформацію або змінюєте налаштування. Без графічного інтерфейсу, без браузера. А ще — AI-агент (наприклад ChatGPT, Claude, Gemini) може робити це замість вас.

---

## 📋 Вимоги

- Node.js 18 або новіший — [завантажити тут](https://nodejs.org/)
- Betaflight польотний контролер, підключений по USB
- macOS / Linux / Windows

# 2. Встановіть залежності

## 🔌 Крок 1: Підключіть контролер та знайдіть порт

Підключіть ваш польотний контролер по USB та виконайте:

```bash
node index.js scan
```

**Приклад виводу:**
```
Available ports:
  /dev/tty.usbmodem0x80000001 | Betaflight | VID:0483
  /dev/tty.Bluetooth-Incoming-Port
```

Порт, де зазначено **Betaflight** або **VID:0483** — це і є ваш контролер.

> **Для AI-агента** (JSON-формат):
> ```bash
> node index.js scan --json
> ```

---

## 🤖 Крок 2: Надайте AI-агенту доступ до контролера

Скопіюйте та надішліть агенту наступне повідомлення:

---

> *Я підключив польотний контролер Betaflight. Для взаємодії з ним використовуй утиліту FlyCLI в директорії `/шлях/до/FlyCLI`.*
>
> *Команди:*
> - *`node index.js scan --json` — знайти порт контролера*
> - *`node index.js execute <PORT> 115200 "<КОМАНДА>" --json` — виконати CLI-команду*
>
> *Порт контролера: `/dev/tty.usbmodem0x80000001`*

---

## ✅ Крок 3: Перевірте підключення

Попросіть AI-агента (або виконайте самостійно):

```bash
node index.js execute /dev/tty.usbmodem0x80000001 115200 "version"
```

Якщо підключення успішне — ви побачите версію прошивки Betaflight.

**Що сказати AI-агенту:**

> *"Перевір підключення до контролера та надай мені: версію прошивки, версію API та назву плати."*

AI виконає команду `version` та поверне результат.

---

## 🛠 Крок 4: Налаштування контролера через AI-агента

Після успішного підключення ви можете попросити агента виконати налаштування.

### Приклади запитів до AI-агента:

**Отримати поточні налаштування:**
> *"Покажи мені всі поточні налаштування контролера (diff all)."*

**Переглянути PID-налаштування:**
> *"Виведи всі PID значення для ролл, піч і яв."*

**Зберегти профіль налаштувань:**
> *"Виконай команду `diff all` та збережи результат у файл `backup.txt`."*

**Скинути до заводських налаштувань:**
> *"Виконай factory reset на контролері."*

## 📖 Довідка по командах

| Команда | Опис |
|---|---|
| `node index.js scan` | Показати всі доступні порти |
| `node index.js scan --json` | Те саме, але у JSON для AI |
| `node index.js execute <PORT> <BAUD> "<CMD>"` | Виконати CLI-команду |
| `node index.js execute ... --json` | Вивести результат у JSON |

### Корисні Betaflight CLI команди:

| CLI Команда | Що робить |
|---|---|
| `version` | Версія прошивки та плати |
| `diff all` | Всі змінені налаштування |
| `dump all` | Повний дамп налаштувань |
| `get pid` | Поточні PID значення |
| `set motor_pwm_rate = 8000` | Змінити частоту PWM двигунів |
| `save` | Зберегти зміни (контролер перезавантажиться) |
| `defaults` | Скинути до заводських налаштувань |

## ⚠️ Важливо

- Команда `save` призводить до **перезавантаження контролера** — це нормально.
- Команда `defaults` **скидає всі налаштування**. Перед нею зробіть `diff all` для резервної копії.
- Не підключайте пропелери під час роботи з USB.

## 💬 Сценарій роботи з AI-агентом: покрокова інструкція

1. **Підключіть контролер** по USB до комп'ютера.
2. **Запустіть сканування**: `node index.js scan`
3. **Скопіюйте порт** (наприклад `/dev/tty.usbmodem0x80000001`).
4. **Відкрийте AI-агента** (ChatGPT, Claude, Gemini тощо).
5. **Надішліть контекст**: вкажіть порт та директорію FlyCLI.
6. **Попросіть агента**: "Перевір версію прошивки на контролері".
7. Агент виконає: `node index.js execute /dev/tty.usbmodem0x80000001 115200 "version" --json`
8. **Отримайте відповідь** та продовжуйте налаштування.
